Description

There is a problem that we faced in the previous stage. We could create vertices only by hardwiring them inside our code. We want our users to interact with our application. So, in this stage, the program will create vertices when users click on the screen.

When users click on the graph, prompt a dialog box JOptionPane that contains a TextBox, an Ok button, and a Cancel button. Take a look at the tutorial by TutorialsPoint to learn more about JOptionPane.

Below you can find the instructions for the three components of the dialog box:

  • TextBox is responsible to obtain a Vertex ID from users;
  • Ok button — clicking on the Ok button closes the dialog box and creates a vertex with the Vertex ID entered by users;
  • Cancel button closes the dialog box.

Make sure that the Vertex ID contains only one character. Prompt a new dialog box if the input is not valid. And the center of each new vertex is created exactly in the position where users make a click.

You can learn about Swing event handling from the article by TutorialsPoint.

For testing reasons, please, do not interact with your computer while tests are running. Also, if you're using macOS, make sure you grant Intellij access to your computer. To do so, follow these steps: go to System Preferences -> Security & Privacy -> Accessibility -> Intellij IDEA CE.app and allow your IDE to control your computer.

Objectives

In this stage, your JFrame should contain:

  1. A graph with the JPanel that represents the graph;
  2. Vertices. Each vertex is represented by a JPanel that is created with a mouse click;
  3. A dialog box (JOptionPane) that is created on the graph with a mouse click;
  4. Name the components as follows:
  • Graph JPanel — Graph;
    • Vertex JPanel — Vertex <id>, for example, Vertex 1, Vertex 2, Vertex A, and so on;
      • Vertex Label JLabel VertexLabel <id>, for example, VertexLabel 1, VertexLabel 2, VertexLabel A, and so on.

Please, maintain the hierarchy above.
