Description

In computer science, graph traversal (also known as graph search) is the process of checking and/or updating each vertex in a graph. A traversal can be classified by the order in which it visits the vertices. Graphs can be traversed in two ways:

  1. Depth-first search (DFS) is an algorithm for traversing or searching tree/graph data structures. The algorithm starts at the root node and explores it as far as possible along each branch before backtracking. In the case of a graph, it can select any node.
  2. Breadth-first search (BFS) algorithm traverses a graph by the breadth and uses a queue to get to the next vertex and to start a search when a dead end occurs in any iteration.
Objectives

To complete this stage, implement the following functionality:

  1. Add the display JLabel to show the results after executing the algorithms;
  2. Add another JMenu named Algorithms that contains two options:
    • Depth-First Search;
    • Breadth-First Search
  3. When any of the two algorithms is selected, the display JLabel should show Please choose a starting vertex;
  4. The selected algorithm starts execution after selecting a starting vertex. Before executing, do the following:
    • Set the current mode to None;
    • The display JLabel should show Please wait...
  5. Feel free with your creative urge. You can add delays to algorithm execution with the swing timer or add some colors to vertices or edges;
  6. After the algorithm execution, change the display JLabel text from Please wait... to the result of the algorithm;
  7. Display the algorithm results in the following format: <Short Algorithm Name> : <The Traversal separated by "->">. For example, "BFS : A -> B -> C -> D -> E -> F", BFS : 1 -> 5 -> 2 -> 3 -> 4 -> 6", "DFS : B -> C -> A -> D -> F -> E". When it comes to choosing edges, choose the lowest weighted edge first.

Tip: Do not forget to reset the color and timers after users select a new algorithm.
