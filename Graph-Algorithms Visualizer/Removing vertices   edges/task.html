Description

Until now users haven't been able to make any changes to graphs. Making changes requires restarting the application. It is not that convenient, isn't it? In this stage, we resolve this issue. Let's add options to remove an edge and a vertex. We will also implement a "soft reset" feature that resets an entire graph.

To implement them, we would require two new menu options: Remove an Edge and Remove a Vertex. If you want to remove an edge, the program needs to remove the edge JComponent and the corresponding edge label (JLabel) from the JPanel graph. As for the vertex removal, a click on a vertex in the Remove a Vertex mode should remove the vertex JPanel together with all the edges.

Objectives

To complete this stage, you need to implement the following:

  1. Add two new JMenuItem options — Remove a Vertex and Remove an Edge;
  2. Add a new File submenu to JMenu. It should contain two JMenuItems:
    • New resets the entire graph. Make sure that no Swing components are present in the panel after clicking on File -> New;
    • Exit closes the frame and exits the application.
  3. Update the JLabel content when the mode is changed;
  4. Implement the Remove a Vertex option according to the procedure mentioned above;
  5. Implement the Remove an Edge according to the procedure mentioned above.

Maintain the hierarchy and naming conventions from the previous stage. As for the new components, name them as follows:

  1. Remove a vertex JMenuItem — Remove a Vertex;
  2. Remove an edge JMenuItem — Remove an Edge;
  3. File JMenu — File;
  4. New JMenuItem — New;
  5. Exit JMenuItem — Exit;
