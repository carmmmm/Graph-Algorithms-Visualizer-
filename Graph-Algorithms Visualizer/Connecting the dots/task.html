Description

Every graph contains edges that connect two vertices. Edges can be of different types: directed and undirected. However, we will deal only with undirected edges in this project. Let's discuss them first.

Directed edges originate from the source vertex and end in the destination vertex, so we can say that it has a direction. It is represented by an arrow (->) where the arrowhead defines its destination. To find more details, please, refer to the Objectives section.

Undirected edges do not have a proper source and destination; each vertex can be considered as the source and destination.

Weighted edges are edges that have some cost. For example, toll roads require a fare while some roads are free. The ones that require a fare can be compared with weighted edges, and the free ones are the not-weighted edges.

We want to give full control to our users, so our application should also have options — let users choose what they want to do, add vertices, edges, remove vertices or edges.

In the Add an Edge mode users can select a vertex when they click on a vertex. Now, if users again click on a different vertex, a dialog box will appear to input the edge weight. After valid input, it will create a new edge between the selected vertices. The first vertex will be the source, and the second vertex is the destination. The edge weight should also be displayed near the edge.

Objectives

In this stage, you need to do the following:

  1. Add a JMenuBar with a JMenu that contains the following options (JMenuItems):
    • Add a Vertex
    • Add an Edge
    • None
  2. Add a JLabel to display the current mode. For example, if Add an Edge is selected, the JLabel would show Add an Edge. The JLabel option is set to Add a Vertex by default. Whenever a new mode is selected, the JLabel must be immediately set to display the new mode;
  3. Add a function to connect two vertices with a JComponent edge. It should be a straight line. For convenience, add two edges (JComponent) — one from the source (the first vertex) to the destination (the second vertex), and the other in the vice-versa direction;
  4. Every edge must have weight. While creating an edge, prompt a dialog box (JOptionPane) for inputting an edge weight that contains an Ok button, a Cancel button, and a Text Box;
  5. The weight of each edge must be displayed in JLabel beside it. Weight is always a digit, update the dialog box every time the weight is input. Make sure that you do not repeat an edge label;
  6. Make sure that the vertices can be added only in the Add a Vertex mode; the edges — only in the Add an Edge mode. In other modes, vertex creating dialog box must not appear.

When creating an edge, adjust the JComponent's bounds so that it will exactly fit the created edge.

Maintain the following hierarchy with the naming conventions:

  • Graph JPanel — Graph
    • Vertex JPanel — Vertex <id>. For example, Vertex 1, Vertex 2, Vertex A, and so on;
      • Vertex label JLabel — VertexLabel <id>. For example, VertexLabel 1, VertexLabel 2, VertexLabel A, and so on;
    • Edge JComponent — Edge <<vId1> -> <vId2>>. For example, Edge <1 -> 2>, Edge <A -> B>, and so on. Edge <1 -> 2> means the edge originates from Vertex 1 and ends at Vertex 2;

    • Edge weight JLabel — EdgeLabel <<vId1> -> <vId2>>. For example, EdgeLabel <1 -> 2>, EdgeLabel <A -> B>, and so on. EdgeLabel <1 -> 2> means the weight of the edge originates from Vertex 1 and ends at Vertex 2;
  • JMenuBar — MenuBar;
    • Mode JMenu — Mode;
      • JMenuItem for adding a vertex — Add a Vertex;
      • JMenuItem for adding an edge — Add an Edge;
      • JMenuItem -> None;
  • JLabel for displaying the current option -> Mode.

If you want, you may add colors for a selected vertex for a better user experience.

Don't implement the functionality for removing vertices and edges. We will embark upon this in the next stage.
